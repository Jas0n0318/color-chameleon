# 🦎 變色龍訓練 — 神經網路色彩感知遊戲

## 專案目的與簡介

透過神經網路理解人類色彩感知，並以此為基礎設計一個「變色龍偽裝」互動遊戲。

使用者先在**訓練實驗室**訓練一個小型神經網路學習 HSV→RGB 的映射關係，然後到**遊戲**中利用對顏色的直覺調整 HSV 滑桿，讓變色龍融入背景色。

核心學習目標：
- 理解 HSV 色彩模型（色相、飽和度、亮度）與 RGB 的關係
- 視覺化神經網路的前向傳遞、訓練曲線與預測誤差
- 在限時壓力下訓練色彩敏感度

技術亮點：
- 純前端，無後端依賴，雙擊 HTML 即可使用（`file://` 相容）
- TensorFlow.js CDN 載入，模型以 localStorage JSON 序列化跨頁面共享
- 完整的神經網路實驗室（資料產生、訓練、曲線、前向傳遞、預測測試）

### 資料流

```
HSV（使用者調整 / 目標顏色）
  │
  ├── hsv2rgb() → RGB（Canvas 繪圖用）
  ├── hsv2hex() → Hex（滑桿光譜顯示用）
  │
  └── deltaE(RGB_user, RGB_target) → 色差值
        ├── 遊戲判定：diff < tolerance → 通過 / 失敗
        └── AI 教練：分析 RGB 比例、色相偏差、飽和度/亮度差異

訓練資料流：
隨機 HSV → hsv2rgb() → extractFeatures() → 8 維特徵向量 → tf.model.fit()
                                                    → predict() → RGB 輸出

模型儲存流：
model.toJSON() + 逐層 getWeights()
  → JSON.stringify({ topology, weights, history })
  → localStorage['color-net-model']
  → 跨頁面共享（training.html ↔ game.html）
```

## 訓練方式

### 目標

讓神經網路學會：**給定一組 HSV 數值，預測對應的 RGB 顏色**。

### 步驟

```
1. 產生隨機 HSV 顏色 (6000 筆，含 10% 灰色)
2. 轉換為 RGB，萃取 8 維特徵向量
3. 建構神經網路（輸入 8 → 輸出 3）
4. 以 (特徵向量, RGB) 配對訓練 40 個 Epoch
5. 每 Epoch 繪製正確率曲線
6. 訓練完成 → 自動儲存至 localStorage
```

### 訓練資料產生

`TrainingDataGenerator.generateUniform(6000)`：

| 步驟 | 細節 |
|------|------|
| 隨機 H | 0~360 均勻分布 |
| 隨機 S | 0.2~1.0（10% 樣本強制 S=0 變成灰色） |
| 隨機 V | 0.2~1.0 |
| 轉換 | `hsv2rgb()` → RGB 作為訓練目標 (Y) |

### 8 維特徵萃取

| # | 特徵 | 公式 | 目的 |
|---|------|------|------|
| 1 | `sin(hueRad)` | `sin(h * PI / 180)` | 色相連續編碼，避免 0°/360° 跳躍 |
| 2 | `cos(hueRad)` | `cos(h * PI / 180)` | 與 sin 搭配唯一確定色相角度 |
| 3 | `saturation` | 直接使用 S 值 | 飽和度資訊 |
| 4 | `value` | 直接使用 V 值 | 亮度資訊 |
| 5 | `chroma` | `max(R,G,B) - min(R,G,B)` | 色彩純度 |
| 6 | `intensity` | `(R+G+B) / 3` | 整體明暗 |
| 7 | `isChromatic` | `chroma > 0.08 ? 1 : 0` | 彩色 vs 灰色分類標記 |
| 8 | `hueBand` | `floor(h / 30) / 12` | 色相量化 12 區段 |

### 神經網路架構

```
輸入層（8 維）—————— 特徵向量
  │
全連接層 · 64 神經元 —— ReLU 激活，576 參數
BatchNorm —————————— 標準化，加速收斂
Dropout 0.15 ———————— 隨機丟棄，防止過擬合
  │
全連接層 · 32 神經元 —— ReLU 激活，2080 參數
BatchNorm —————————— 標準化
Dropout 0.10 ———————— 隨機丟棄
  │
全連接層 · 16 神經元 —— ReLU 激活，544 參數
  │
輸出層 · 3 神經元 ——— Sigmoid 激活 → (R, G, B) ∈ [0, 1]
```

| 項目 | 值 |
|------|-----|
| 總參數 | ~3,200 |
| 優化器 | Adam (lr = 0.001) |
| 損失函數 | MSE（均方誤差） |
| 評估指標 | MAE（平均絕對誤差） |
| Batch Size | 128 |
| Epochs | 40 |
| 驗證集比例 | 15% |

### 正確率計算

```
accuracy(%) = max(0, 1 - MAE / 0.375) × 100
```

- 隨機猜測時 MAE ≈ 0.375 → 正確率 ≈ 0%
- MAE = 0.0375 → 正確率 ≈ 90%
- MAE = 0.01875 → 正確率 ≈ 95%
- MAE → 0 → 正確率 → 100%

### 模型儲存與載入

不使用 TF.js 內建 `localstorage://` / `indexeddb://`（在 `file://` 被瀏覽器阻擋）。

手動序列化流程：

```
儲存（training.html）：
  model.toJSON() → 取得模型拓樸
  + 逐層 layer.getWeights() → { shape, data[] }
  → JSON.stringify({ topology, weights, history })
  → localStorage.setItem('color-net-model', ...)

載入（game.html）：
  localStorage.getItem('color-net-model') → JSON.parse
  → tf.models.modelFromJSON(topology) 重建模型
  → layer.setWeights(tensors) 還原權重
  → model.compile() 設定優化器與損失函數

資料不存在時（尚未訓練）：
  hasSavedData() → false → 遊戲顯示「尚未訓練模型」遮罩

TF.js 不可用時：
  loadModelFromStorage() 失敗 → 遊戲正常啟動
  → 教練側欄顯示「⚠️ AI 模型未載入」
  → 核心遊戲功能完全不受影響
```

## 遊戲方式

### 概述

玩家扮演一隻變色龍，在 5 個關卡中逐一將自己的體色調整至與背景一致。每關限時 45 秒，時間到自動判定。滑桿調整過程中 AI 教練**不會即時回應**，只有按下「確認」或時間結束後才顯示分析結果。

### 關卡結構

| 關卡 | 名稱 | 容許色差 (Tolerance) | 難度 |
|------|------|----------------------|------|
| 1 | 第1關 | 0.20 | 寬鬆，熟悉操作 |
| 2 | 第2關 | 0.15 | 略緊 |
| 3 | 第3關 | 0.12 | 中等 |
| 4 | 第4關 | 0.08 | 需要精準 |
| 5 | 第5關 | 0.05 | 幾乎完美 |

- **容許值** = DeltaE（RGB 歐氏距離），值越小越精準
- **所有關卡通過** = 獲得勝利

### 色差計算

```
DeltaE = √[(R₁−R₂)² + (G₁−G₂)² + (B₁−B₂)²]
```

RGB 值範圍 0~1。例如目標 `(0.8, 0.2, 0.3)`、使用者 `(0.7, 0.3, 0.3)`，誤差約 0.14。

### 完整流程

```
開始遊戲
  │
  ▼
第 N 關開始（N=1 → 5）
  │
  ├── Canvas 純色背景（目標色）
  ├── 左側目標變色龍 / 右側你的變色龍
  ├── HUD：第N關 / 倒數計時 45 秒
  │
  ├── 玩家調整 H/S/V 滑桿
  │     └── 無即時教練回饋
  │
  ├── [手動] 按下「✓ 確認」→ 立即判定
  │
  └── [自動] 時間到 (45s) → 自動判定
        │
        ▼
     計算 DeltaE(diff, tol)
        │
   ┌────┴────┐
   │         │
 通過       失敗
 (diff<tol) (diff≥tol)
   │         │
   │         ├── Canvas 遮罩「偽裝失敗」
   │         ├── AI 教練顯示分析（通道偏差、色相偏移、飽和度/亮度建議）
   │         └──「返回訓練」按鈕 → training.html
   │
   ├── N < 5 → Canvas 遮罩「第N關通過！」→ 點擊繼續
   │
   └── N = 5 → 勝利畫面「🎉 全部通關！」→ 返回訓練頁
```

### 操作方式

| 操作 | 說明 |
|------|------|
| H 滑桿 (0°–360°) | 調整色相（紅橙黃綠藍紫），滑桿背景為彩虹光譜 |
| S 滑桿 (0%–100%) | 調整飽和度（灰 → 鮮豔），背景為目前色相的灰→純色漸層 |
| V 滑桿 (0%–100%) | 調整亮度（暗 → 亮），背景為目前色相+飽和度的黑→亮漸層 |
| ✓ 確認 | 立即檢查顏色是否通過容許範圍 |
| ▶ 繼續 | 回合結束後前進下一關 |
| 點擊畫布 | 同「繼續」按鈕 |
| ▲ 收合 / ▼ 展開 | 切換 AI 教練側欄 |

### AI 教練回饋內容

僅在判定後顯示於側欄：

```
✓ 通過！誤差 X.X% 在容許範圍內

✗ 色差過大
誤差 X.X% / 容許 X.X%
• 🔴 紅色比例不足 (目標XX% : 目前XX%)
• 🔄 色相差 XX°，請將色相順時針調整
• 🎨 飽和度偏低 (目標XX% : 目前XX%)
• ☀️ 亮度不足 (目標XX% : 目前XX%)
```

### 結束條件

| 情境 | 結果 |
|------|------|
| 任一關卡色差 ≥ 容許值 | 遊戲結束 → AI 教練分析 → 返回訓練頁 |
| 通過全部 5 關 | 勝利畫面 → 返回訓練頁 |
| TF.js 未載入 | 遊戲正常進行，教練顯示「⚠️ AI 模型未載入」，無深度分析 |
